# Contributing Guidelines

## Overview

This document provides streamlined guidelines for contributing to the NaNLABS Data Warehouse platform. For comprehensive contribution information, please refer to the main [CONTRIBUTING.md](../CONTRIBUTING.md) document.

### Key Principles
- **Documentation-First**: Always reference the `docs/` directory before implementation
- **Security**: Never expose sensitive data or credentials
- **Traceability**: Maintain clear audit trails and documentation
- **Quality**: Follow established coding standards and testing requirements

## Development Workflow

1. **Review Documentation**: Check relevant documentation in `docs/`
2. **Create Task**: Use ClickUp for task tracking and management
3. **Follow Guidelines**: Adhere to AI guidelines in [AGENTS.md](../AGENTS.md) when using AI assistance
4. **Implement Changes**: Follow best practices and coding standards
5. **Test Thoroughly**: Include unit and integration tests
6. **Document Updates**: Update relevant documentation as needed
7. **Submit PR**: Follow pull request guidelines in main contributing document

## Important Notes

- **Single Source of Truth**: The `docs/` directory contains all authoritative documentation
- **No Duplication**: Avoid creating documentation outside the `docs/` directory
- **AI Assistance**: Follow guidelines in [AGENTS.md](../AGENTS.md) when using AI tools
- **Project Management**: Use ClickUp for all task and project tracking
